## Objetivo

Aprender a usar LINQ para consultar y transformar colecciones en C# de forma declarativa, legible y reutilizable.

## Prerrequisitos

- Saber crear listas y objetos en C#.
- Entender lambdas como `x => x > 10`.
- Tener presente que LINQ vive en `System.Linq`.

```csharp
using System.Linq;
```

## Qué es LINQ

LINQ significa `Language Integrated Query`. Permite consultar datos con una sintaxis uniforme sobre:

- listas en memoria
- arrays
- diccionarios
- resultados de bases de datos
- XML y otras fuentes

La idea central es escribir **qué** querés obtener, no **cómo** iterar paso a paso.

```csharp
var nombres = personas
    .Where(p => p.Edad >= 18)
    .OrderBy(p => p.Apellido)
    .Select(p => p.Nombre)
    .ToList();
```

## Datos de ejemplo

Vamos a usar una colección simple para recorrer las operaciones principales.

```csharp
record Empleado(int Id, string Nombre, string Area, int Edad, decimal Sueldo);

var empleados = new List<Empleado>
{
    new(1, "Ana", "Ventas", 28, 1200m),
    new(2, "Luis", "IT", 35, 2200m),
    new(3, "Mora", "IT", 31, 2100m),
    new(4, "Juan", "RRHH", 42, 1800m),
    new(5, "Lara", "Ventas", 25, 1250m),
};
```

## 1. Sorting

El video arranca con ordenamiento. LINQ ofrece métodos para ordenar por una o varias claves.

### `OrderBy`

```csharp
var porNombre = empleados.OrderBy(e => e.Nombre);
```

### `OrderByDescending`

```csharp
var porSueldoDesc = empleados.OrderByDescending(e => e.Sueldo);
```

### `ThenBy` y `ThenByDescending`

```csharp
var ordenados = empleados
    .OrderBy(e => e.Area)
    .ThenByDescending(e => e.Sueldo);
```

Usá `OrderBy` para la primera clave y `ThenBy` para las siguientes.

## 2. Set Operations

Estas operaciones tratan las colecciones como conjuntos.

### `Distinct`

```csharp
var areas = empleados.Select(e => e.Area).Distinct();
```

### `Union`

```csharp
var a = new[] { 1, 2, 3 };
var b = new[] { 3, 4, 5 };

var union = a.Union(b); // 1, 2, 3, 4, 5
```

### `Intersect`

```csharp
var interseccion = a.Intersect(b); // 3
```

### `Except`

```csharp
var diferencia = a.Except(b); // 1, 2
```

Estas operaciones son útiles cuando querés comparar colecciones.

## 3. Filtering

Filtrar es probablemente la operación más usada en LINQ.

### `Where`

```csharp
var empleadosIT = empleados.Where(e => e.Area == "IT");
var mayoresDe30 = empleados.Where(e => e.Edad > 30);
```

Podés combinar condiciones.

```csharp
var itBienPago = empleados.Where(e => e.Area == "IT" && e.Sueldo > 2000m);
```

## 4. Quantifier Operations

Estas operaciones responden preguntas sobre la colección.

### `Any`

```csharp
bool hayAlguienEnRRHH = empleados.Any(e => e.Area == "RRHH");
```

### `All`

```csharp
bool todosMayoresDeEdad = empleados.All(e => e.Edad >= 18);
```

### `Contains`

```csharp
var ids = empleados.Select(e => e.Id);
bool existeEl3 = ids.Contains(3);
```

`Any` pregunta si existe al menos uno. `All` pregunta si todos cumplen.

## 5. Projection

Proyectar significa transformar cada elemento en otra forma.

### `Select`

```csharp
var nombres = empleados.Select(e => e.Nombre);
```

También podés crear objetos nuevos:

```csharp
var resumen = empleados.Select(e => new
{
    e.Nombre,
    e.Area
});
```

### `SelectMany`

Cuando cada elemento contiene otra colección, `SelectMany` aplana el resultado.

```csharp
var palabras = new[]
{
    "LINQ es poderoso",
    "C# es expresivo"
};

var todasLasPalabras = palabras.SelectMany(t => t.Split(' '));
```

## 6. Partitioning

Permiten tomar o saltear partes de una colección.

### `Take`

```csharp
var primerosDos = empleados.Take(2);
```

### `Skip`

```csharp
var desdeElTercero = empleados.Skip(2);
```

### `TakeWhile`

```csharp
var menoresDe40 = empleados.TakeWhile(e => e.Edad < 40);
```

### `SkipWhile`

```csharp
var saltearJovenes = empleados.SkipWhile(e => e.Edad < 30);
```

`Take` y `Skip` son muy comunes en paginación.

## 7. Joins

Los joins relacionan dos colecciones por una clave común.

```csharp
record Proyecto(int Id, int EmpleadoId, string Nombre);

var proyectos = new List<Proyecto>
{
    new(1, 2, "Migración"),
    new(2, 2, "API"),
    new(3, 3, "Panel Admin"),
    new(4, 5, "Campaña")
};
```

### `Join`

```csharp
var empleadoProyecto = empleados.Join(
    proyectos,
    e => e.Id,
    p => p.EmpleadoId,
    (e, p) => new { e.Nombre, Proyecto = p.Nombre }
);
```

Esto equivale a un inner join.

## 8. Grouping

`GroupBy` agrupa elementos por una clave.

```csharp
var porArea = empleados.GroupBy(e => e.Area);

foreach (var grupo in porArea)
{
    Console.WriteLine(grupo.Key);
    foreach (var empleado in grupo)
        Console.WriteLine($"- {empleado.Nombre}");
}
```

También se puede combinar con agregaciones:

```csharp
var cantidadPorArea = empleados
    .GroupBy(e => e.Area)
    .Select(g => new
    {
        Area = g.Key,
        Cantidad = g.Count(),
        SueldoPromedio = g.Average(e => e.Sueldo)
    });
```

## 9. Generation

LINQ también genera secuencias.

### `Range`

```csharp
var numeros = Enumerable.Range(1, 5); // 1, 2, 3, 4, 5
```

### `Repeat`

```csharp
var ceros = Enumerable.Repeat(0, 3); // 0, 0, 0
```

### `Empty`

```csharp
var vacio = Enumerable.Empty<string>();
```

Estas secuencias son útiles para pruebas, inicialización o composición.

## 10. Element Operations

Sirven para obtener elementos puntuales.

### `First` y `FirstOrDefault`

```csharp
var primeroIT = empleados.First(e => e.Area == "IT");
var nadie = empleados.FirstOrDefault(e => e.Nombre == "Zoe");
```

### `Last` y `LastOrDefault`

```csharp
var ultimo = empleados.Last();
```

### `Single` y `SingleOrDefault`

```csharp
var empleadoId2 = empleados.Single(e => e.Id == 2);
```

Usá `Single` solo cuando esperás exactamente un resultado.

## 11. Concatenation

### `Concat`

```csharp
var equipoA = new[] { "Ana", "Luis" };
var equipoB = new[] { "Mora", "Juan" };

var todos = equipoA.Concat(equipoB);
```

`Concat` une secuencias respetando el orden y sin eliminar duplicados.

## 12. Aggregation

Acá LINQ resume una colección en un solo valor.

### `Count`

```csharp
int cantidad = empleados.Count();
```

### `Sum`

```csharp
decimal totalSueldos = empleados.Sum(e => e.Sueldo);
```

### `Min` y `Max`

```csharp
decimal minimo = empleados.Min(e => e.Sueldo);
decimal maximo = empleados.Max(e => e.Sueldo);
```

### `Average`

```csharp
decimal promedio = empleados.Average(e => e.Sueldo);
```

### `Aggregate`

`Aggregate` es la forma más general de acumulación.

```csharp
var csv = empleados
    .Select(e => e.Nombre)
    .Aggregate((acc, nombre) => acc + ", " + nombre);
```

## 13. Ejemplos prácticos del cierre del video

En la parte final, el video pasa de operaciones clásicas a mini ejercicios aplicados. La idea importante no es memorizar cada caso, sino ver que LINQ sirve también para texto, archivos y procesamiento de datos cotidianos.

### Buscar ocurrencias de palabras en un texto

```csharp
string texto = "linq permite consultar datos con linq de forma declarativa";

var frecuencia = texto
    .Split(' ', StringSplitOptions.RemoveEmptyEntries)
    .GroupBy(p => p.ToLower())
    .Select(g => new { Palabra = g.Key, Cantidad = g.Count() })
    .OrderByDescending(x => x.Cantidad);
```

### Extraer números desde texto

```csharp
string linea = "Pedido 123, cliente 45, total 980";

var numerosExtraidos = linea
    .Split(new[] { ' ', ',', ';' }, StringSplitOptions.RemoveEmptyEntries)
    .Where(token => int.TryParse(token, out _))
    .Select(int.Parse);
```

### Filtrar archivos por extensión

```csharp
var archivos = Directory.GetFiles("/ruta/carpeta");

var pdfs = archivos.Where(a => Path.GetExtension(a).Equals(".pdf", StringComparison.OrdinalIgnoreCase));
```

### Obtener el archivo más grande

```csharp
var archivoMasGrande = new DirectoryInfo("/ruta/carpeta")
    .GetFiles()
    .OrderByDescending(f => f.Length)
    .FirstOrDefault();
```

## 14. Orden mental para leer una consulta LINQ

Una consulta encadenada suele leerse bien en este orden:

1. fuente de datos
2. filtro
3. orden
4. transformación
5. materialización

```csharp
var resultado = empleados
    .Where(e => e.Sueldo > 1500m)
    .OrderBy(e => e.Nombre)
    .Select(e => new { e.Nombre, e.Sueldo })
    .ToList();
```

## 15. Cuándo materializar

Muchos métodos LINQ usan evaluación diferida. Eso significa que la consulta no se ejecuta hasta recorrerla.

```csharp
var consulta = empleados.Where(e => e.Area == "IT");
```

Si necesitás ejecutar ya y guardar el resultado, materializá:

```csharp
var lista = consulta.ToList();
var arreglo = consulta.ToArray();
```

## 16. Errores comunes

### Usar `Single` cuando puede haber varios

```csharp
var item = empleados.Single(e => e.Area == "IT"); // falla si hay más de uno
```

### Olvidar materializar antes de mutar la fuente

Si cambiás la colección original antes de iterar, la consulta puede dar resultados distintos.

### Encadenar demasiado sin nombrar pasos intermedios

Si una consulta queda larga, conviene separar.

```csharp
var filtrados = empleados.Where(e => e.Sueldo > 1500m);
var ordenados = filtrados.OrderBy(e => e.Area);
var resultado = ordenados.Select(e => e.Nombre).ToList();
```
