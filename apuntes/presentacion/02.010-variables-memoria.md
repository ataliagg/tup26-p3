# Variables y Manejo de Memoria en C#

Entendiendo el modelo mental de las variables

---

## ¿Qué es una variable?

Una variable es un nombre asociado a un dato que el programa puede usar durante su ejecución.

**Tipo**: Determina:
- Qué valores son válidos
- Qué operaciones se pueden hacer
- Cuánto espacio suele ocupar
- Cómo interpreta el compilador

```cs
int edad = 20;
string nombre = "Ana";
bool activo = true;
```

---

## C# como lenguaje estático y tipado fuerte

**Tipado estático**: El tipo de cada variable se conoce antes de ejecutar.

```cs
int edad = 20;
edad = 30;      // ✓ válido
edad = "treinta";  // ✗ error en compilación
```

**Tipado fuerte**: No mezcla tipos incompatibles ni permite conversiones peligrosas.

```cs
int numero = 10;
string texto = "20";
numero = texto;  // ✗ error en compilación
numero = int.Parse(texto);  // ✓ conversión explícita
```

---

## Sintaxis básica y asignación

```cs
// Declaración e inicialización
int edad = 20;
string nombre = "Ana";

// Declarar sin inicializar
int cantidad;
cantidad = 10;

// Asignación y actualización
int contador = 0;
contador = contador + 1;  // 1
contador += 1;            // 2
```

Una variable puede cambiar de valor, pero no de tipo.

---

## `var` e inferencia de tipos

`var` no significa "tipo dinámico". El compilador **infiere** el tipo:

```cs
var edad = 20;        // int
var ciudad = "Rosario";  // string
var precio = 19.99;   // double
```

**Cuándo usarlo:**
- ✓ Cuando el tipo es evidente
- ✗ Cuando necesita claridad

```cs
var alumnos = new List<string>();  // claro
var dato = ObtenerResultado();      // confuso
```

---

## Constantes y `readonly`

**`const`**: Valor fijo conocido en compilación

```cs
const double Pi = 3.14159;
const int MaxTamaño = 100;
```

**`readonly`**: Se fija una vez al construir

```cs
class Persona {
    public readonly DateTime FechaCreacion;
    
    public Persona() {
        FechaCreacion = DateTime.Now;
    }
}
```

---

## Alcance y tiempo de vida

**Alcance**: Desde qué partes del código es accesible

```cs
if (true) {
    int numero = 5;
    Console.WriteLine(numero);  // ✓ aquí existe
}
// Console.WriteLine(numero);  // ✗ aquí no existe
```

**Tiempo de vida**: Cuánto existe en memoria

- Variable local: mientras ejecuta su bloque
- Campo de objeto: mientras vive el objeto
- Dato estático: durante toda la aplicación

---

## Stack vs Heap

- **Stack**: Variables locales, administración rápida y automática
- **Heap**: Objetos dinámicos, administrados por recolector de basura

La idea clave: no toda la memoria se comporta igual.

---

## Tipos por valor vs tipos por referencia

**Tipos por valor**: Se copia el dato

```cs
int a = 10;
int b = a;
b = 20;
Console.WriteLine(a);  // 10 — sin cambios
```

**Tipos por referencia**: Se copia la referencia

```cs
int[] numeros = { 1, 2, 3 };
int[] copia = numeros;
copia[0] = 99;
Console.WriteLine(numeros[0]);  // 99 — cambio visible
```

---

## Namespaces y organización

**Namespace**: Agrupa tipos relacionados

```cs
namespace MiAplicacion.Utilidades;
```

**Clases estáticas**: Agrupan miembros utilitarios

```cs
static class ConstantesDelSistema {
    public const int EdadMinima = 18;
    public static readonly string MensajeBienvenida = "Bienvenido";
}
```

---

## Recolección de basura

El runtime de .NET libera automáticamente objetos sin referencias activas.

**Importante:**
- No es en el instante exacto en que se pierden referencias
- El recolector **puede** recuperar memoria, no **debe** hacerlo de inmediato
- Los objetos inaccesibles pueden ser recolectados cuando el runtime lo considere conveniente
