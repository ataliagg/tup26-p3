# Null y Tipos Anulables en C#

Manejando la ausencia de valor correctamente

---

## Qué es `null`

`null` representa la **ausencia de un valor** o de un objeto, no es cero ni vacío ni falso.

Mal manejado causa el error más frecuente:
```
NullReferenceException: Object reference not set to an instance of an object.
```

---

## Quién puede tener `null`

**Tipos por referencia** — siempre pueden ser `null`:
```csharp
string nombre = null;   // ✓ válido
int[] datos = null;     // ✓ válido
```

**Tipos por valor** — por defecto NO pueden ser `null`:
```csharp
int edad = null;        // ✗ error
bool flag = null;       // ✗ error
```

Para que acepten `null`, usar **nullable value types** con `T?`.

---

## Nullable value types — `T?`

```csharp
int? edad = null;           // puede ser int o null
double? precio = null;
bool? confirmado = true;    // o null

// Propiedades
if (edad.HasValue) {
    Console.WriteLine(edad.Value);  // acceso seguro
}

// Métodos seguros
int valor = edad.GetValueOrDefault();      // 0 si null
int conDefault = edad.GetValueOrDefault(-1);  // -1 si null
```

---

## Comparaciones con nullable

```csharp
int? a = 5;
int? b = null;

Console.WriteLine(a == 5);      // true
Console.WriteLine(b == null);   // true
Console.WriteLine(a > b);       // false — null en comparación da false
```

---

## Nullable reference types (C# 8+)

Distingue entre referencias que pueden y no pueden ser `null`:

```csharp
string nombre = "Ana";       // nunca null (el compilador lo garantiza)
string? apellido = null;     // puede ser null (nullable reference type)

string nombre2 = null;       // Warning — el compilador advierte
```

---

## Análisis de flujo estático

El compilador sabe cuándo una variable puede ser `null`:

```csharp
string? texto = ObtenerTexto();

// Warning — posible null
Console.WriteLine(texto.Length);

// ✓ Aquí el compilador sabe que no es null
if (texto != null) {
    Console.WriteLine(texto.Length);
}

// Pattern matching
if (texto is not null) {
    Console.WriteLine(texto.Length);
}
```

---

## Operador `?.` — Acceso condicional

Accede a un miembro **solo si el objeto no es null**:

```csharp
string? nombre = null;

// Sin ?. — NullReferenceException
int largo = nombre.Length;   // 💥

// Con ?. — retorna null si nombre es null
int? largo = nombre?.Length;  // null, sin excepción
```

Puede encadenarse:

```csharp
Pedido? pedido = ObtenerPedido();
Cliente? cliente = pedido?.Cliente;
string? ciudad = pedido?.Cliente?.Direccion?.Ciudad;
```

También con métodos:

```csharp
string? upper = nombre?.ToUpper();
char? primera = nombre?[0];
```

---

## Operador `??` — Coalescencia nula

Devuelve la izquierda si no es `null`, o la derecha si lo es:

```csharp
string? nombre = null;
string texto = nombre ?? "Anónimo";  // "Anónimo"

int? edad = null;
int valor = edad ?? 0;  // 0

// Encadenado
string resultado = a ?? b ?? c;  // primer no-null
```

---

## Operador `??=` — Asignación de coalescencia

Asigna solo si la variable **es null**:

```csharp
string? cache = null;
cache ??= "valor por defecto";   // se asigna
cache ??= "otro valor";          // no cambia — ya tiene valor

// Útil para inicialización perezosa
List<string>? _items = null;

public List<string> Items {
    get {
        _items ??= new List<string>();
        return _items;
    }
}
```

---

## Operador `!` — Null-forgiving

Le dice al compilador "confío en que esto no es null":

```csharp
string? texto = ObtenerTexto();

// Sin ! — Warning
int largo = texto.Length;

// Con ! — sin warning (pero es responsabilidad tuya)
int largo = texto!.Length;  // si es null, excepción en runtime
```

⚠️ Usarlo solo cuando realmente estés seguro.

---

## Pattern matching con null

```csharp
string? nombre = ObtenerNombre();

// is null / is not null
if (nombre is null) {
    Console.WriteLine("No hay nombre");
}

if (nombre is not null) {
    Console.WriteLine($"Hola, {nombre}");
}

// En switch
string respuesta = nombre switch {
    null => "No ingresaste nada",
    "" => "Ingresaste vacío",
    { Length: > 20 } => "Demasiado largo",
    _ => $"Ingresaste: {nombre}"
};
```

---

## Combinando operadores

```csharp
record Cliente(string Nombre, Direccion? Direccion);
record Direccion(string Ciudad);

Cliente? cliente = ObtenerCliente();

// Acceso profundo seguro
string? ciudad = cliente?.Direccion?.Ciudad;

// Con fallback
string display = cliente?.Direccion?.Ciudad ?? "Ciudad desconocida";

// En validación
if (cliente?.Direccion?.Ciudad is not null) {
    Console.WriteLine($"Cliente en {cliente.Direccion.Ciudad}");
}
```

## Métodos de validación

```csharp
// Validar y lanzar excepción
static void Procesar(string nombre) {
    ArgumentNullException.ThrowIfNull(nombre);
    ArgumentException.ThrowIfNullOrWhiteSpace(nombre);
    // A partir de aquí, el compilador sabe que no es null
}

// Para strings
string.IsNullOrEmpty(texto);       // ✓ si es null o ""
string.IsNullOrWhiteSpace(texto);  // ✓ si es null, "" o "   "
```

## Buenas prácticas

**Preferir `is null` sobre `== null`:**
```csharp
if (objeto is null) { ... }     // ✓ mejor
if (objeto == null) { ... }     // válido pero puede ser sobrecargado
if (objeto is not null) { ... } // ✓ recomendado
```

**Validar parámetros al inicio:**
```csharp
public void Procesar(string entrada) {
    ArgumentNullException.ThrowIfNull(entrada);
    // Ahora el compilador sabe que entrada no es null
    Console.WriteLine(entrada.ToUpper());
}
```

**No abusar del `!`:**
```csharp
// ✗ silencia pero no resuelve
string resultado = ObtenerTexto()!.ToUpper();

// ✓ manejar explícitamente
string resultado = ObtenerTexto() is not null
    ? ObtenerTexto().ToUpper()
    : string.Empty;
```

## Nullable en colecciones

```csharp
// Lista que puede ser null
List<string>? lista = null;

// Lista de strings que pueden ser null
List<string?> nombres = ["Ana", null, "Carlos"];

// Filtrar nulls
List<string> validos = nombres
    .Where(n => n is not null)
    .Select(n => n!)
    .ToList();

// Con OfType<T> — filtra y castea automáticamente
List<string> soloValidos = nombres.OfType<string>().ToList();
```

---

## Resumen de operadores

| Operador | Uso                                    |
|----------|----------------------------------------|
| `?.`     | Acceso condicional a miembros         |
| `??`     | Valor de fallback si es null          |
| `??=`    | Asignar solo si es null               |
| `!`      | Suprimir warning (null-forgiving)    |
| `is null` | Verificar si es null                  |
| `is not null` | Verificar si NO es null             |
